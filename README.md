# Компіляція скінченного автомату для regex-виразу
Цей репозиторій містить реалізацію лабораторної роботи із дискретної математики.

## Коротке пояснення імплементації

RegexFSM компілює regex-вираз у граф станів, читаючи патерн посимвольно. Кожен символ створює відповідний стан: AsciiState приймає конкретний символ, DotState — будь-який, а квантифікатори * і + огортають попередній стан у StarState або PlusState. Під час побудови підтримується frontier — список станів із «відкритим кінцем» графу, до яких ще не підключено наступника. Для * frontier розширюється назад до батьківських станів (щоб стан можна було пропустити — нуль разів), для + frontier залишається лише сам PlusState (бо мінімум одне входження обов'язкове). В кінці до всіх станів frontier підключається TerminationState.

Перевірка рядка у check_string — це просто обхід цього графу: поточний стан викликає check_next(char), що шукає у своєму списку next_states перший стан, який приймає символ, і повертає його. StarState та PlusState перевизначають check_next з особливою логікою: спочатку намагаються вийти з циклу (перевіряють інші стани), і лише якщо не вийшло — залишаються в собі. Якщо після обробки всього рядка поточний стан є TerminationState або має його серед наступників — рядок прийнятий, інакше — ні.

## Інструкція до запуску

Ви можете використовувати цей проєкт як бібліотеку у своєму коді або як консольну програму через argparse.

### 1. Використання як модуля

Імпортуйте клас у свій Python-скрипт для програмної перевірки рядків:

```python
from regex import RegexFSM

# Ініціалізація автомата
regex_pattern = "a*b+"
fsm = RegexFSM(regex_pattern)

# Перевірка рядка
test_string = "aaabb"
print(fsm.check_string(test_string))
```

### 2. Використання через консоль

Приклад запуску у консолі:

python regex.py <regex-expr> <string_to_check>

```python
python regex.py "a*b" "aaab"
```
